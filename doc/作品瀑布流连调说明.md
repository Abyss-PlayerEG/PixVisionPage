# 作品瀑布流组件联调说明

## 文件结构

```
src/
├── config/
│   └── api.js                    # API 配置（已添加作品相关接口）
├── api/
│   └── workApi.js                # 作品 API 调用（新建）
├── composables/
│   └── useWorkWaterfall.js       # 作品瀑布流业务逻辑（新建）
└── views/
    └── mainIndex.vue             # 主页视图（已更新）
```

## 数据流程

### 1. API 层 (`src/api/workApi.js`)

**获取作品分页数据：**
```javascript
fetchWorkPage(current, size)
// 返回格式：{ success: boolean, data: Object, message: string }
```

**数据转换函数：**
```javascript
transformWorksToWaterfallFormat(records)
// 将后端 records 转换为瀑布流组件所需格式
// 返回：[{ src, alt, height, workId, workTitle, userId }]
```

### 2. Composable 层 (`src/composables/useWorkWaterfall.js`)

**提供的功能：**
- `waterfallImages`: 响应式图片数据数组
- `isLoading`: 加载状态
- `error`: 错误信息
- `loadWorks(page, size)`: 加载指定页数据
- `loadMoreWorks(page, size)`: 加载更多数据（追加）

**自动加载：**
- 组件挂载时自动加载第 1 页，每页 50 条

### 3. 视图层 (`src/views/mainIndex.vue`)

**使用方式：**
```vue
<Waterfall 
  :external-images="waterfallImages" 
  :pin-offset-top="70"
/>
```

## 接口说明

### 作品列表接口

**请求：**
```
GET /api/work/page/{current}/{size}
```

**注意：**
- 使用 GET 方法，不是 POST
- URL 路径参数直接拼接在 URL 中

**响应示例：**
```json
{
  "data": {
    "records": [
      {
        "work_id": 1484,
        "user_id": 1,
        "work_title": "【树】二叉叔叔493",
        "img_url": "f3658b6a14624c6bab6f391752dd72db.jpg",
        ...
      }
    ],
    "total": 1484,
    "size": 1,
    "current": 1,
    "pages": 1484
  },
  "message": "查询成功",
  "recode": 200
}
```

### 作品图片接口

**请求：**
```
GET /api/image/work/get?filePath={img_url}
```

**参数：**
- `filePath`: 作品图片路径（从 `img_url` 字段获取）

## 常见问题

### Q1: 405 Method Not Allowed 错误？

**原因：**
- 请求方法不正确（后端要求 GET，但使用了 POST）
- URL 出现双斜杠（如 `http://localhost:9090//api/...`）

**解决方案：**
1. 确认使用 GET 方法
2. 检查 `API_BASE_URL` 末尾不要有斜杠 `/`
   ```javascript
   // 正确
   export const API_BASE_URL = 'http://localhost:9090';
   
   // 错误
   export const API_BASE_URL = 'http://localhost:9090/';
   ```

### Q2: 图片不显示？

**检查步骤：**
1. 确认后端服务是否启动（http://localhost:9090）
2. 检查 Network 面板是否有失败的图片请求
3. 确认 `img_url` 字段是否正确
4. 验证图片接口返回的图片数据

### Q3: 瀑布流没有水平滚动效果？

**可能原因：**
1. 图片数量太少（需要足够多的图片才能触发水平滚动）
2. 容器宽度不够
3. ScrollTrigger 未正确初始化

**解决方案：**
- 增加每页加载数量（修改 `loadWorks(1, 100)`）
- 检查控制台是否有 GSAP 相关错误

### Q4: 如何加载更多数据？

**当前实现：**
- 自动加载第 1 页 50 条数据

**扩展方案：**
```javascript
// 在 composable 中添加滚动监听
import { onMounted, onUnmounted } from 'vue'

let currentPage = 1

const handleScroll = () => {
  const scrollHeight = document.documentElement.scrollHeight
  const scrollTop = document.documentElement.scrollTop
  const clientHeight = document.documentElement.clientHeight
  
  // 距离底部 200px 时加载更多
  if (scrollHeight - scrollTop - clientHeight < 200 && !isLoading.value) {
    currentPage++
    loadMoreWorks(currentPage, 50)
  }
}

onMounted(() => {
  window.addEventListener('scroll', handleScroll)
})

onUnmounted(() => {
  window.removeEventListener('scroll', handleScroll)
})
```

## 性能优化建议

### 1. 图片懒加载

瀑布流组件已内置 `loading="lazy"` 属性，浏览器会自动懒加载。

### 2. 分页加载

建议每页加载 50-100 条数据，避免一次性加载过多。

### 3. 虚拟滚动

如果数据量非常大（>500 条），考虑实现虚拟滚动，只渲染可视区域的图片。

### 4. 图片压缩

确保后端返回的图片已经过压缩处理，建议使用 WebP 格式。
