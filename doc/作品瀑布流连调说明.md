# 作品瀑布流组件联调与使用说明书

## 1. 概述
本瀑布流组件采用 **Composable + Component** 的架构设计，支持通过 JS 极简调用。它集成了智能高度分配算法，并深度对接后端双接口（分页信息 + 图像获取）。

### 核心特性
- **JS 驱动**：像 `NotificationToast` 一样，通过 Composable 即可控制数据加载。
- **双接口集成**：自动处理 `/api/work/page` 和 `/api/image/work/get` 的拼接。
- **智能布局**：内置“周期性交换 + 留白补偿”算法，确保视觉美感。
- **高性能**：支持骨架屏、懒加载及 GSAP 水平滚动动画。

## 2. 快速开始 (Quick Start)

### 2.1 基础用法（在任意页面）

```vue
<script setup>
import Waterfall from '@/components/Waterfall.vue'
import { useWorkWaterfall } from '@/composables/useWorkWaterfall'

// 1. 实例化 Composable
const { waterfallImages, loadWorks, isLoading } = useWorkWaterfall()

// 2. 在需要时加载数据（如 onMounted 或点击事件）
const handleLoad = () => {
  loadWorks({ reset: true }) // 重置并加载第一页
}
</script>

<template>
  <button @click="handleLoad" :disabled="isLoading">
    {{ isLoading ? '加载中...' : '加载作品' }}
  </button>
  
  <!-- 3. 绑定数据 -->
  <Waterfall 
    :external-images="waterfallImages" 
    :pin-offset-top="70"
  />
</template>
```

### 2.2 高级用法（带筛选条件）

```javascript
const { loadWorks } = useWorkWaterfall()

// 加载特定用户的原创作品
loadWorks({
  reset: true,
  filters: {
    userId: 123,
    isOriginal: true,
    workTitle: '风景' // 支持模糊查询
  }
})
```

## 3. API 接口说明

### 3.1 图像信息分页获取
**接口地址**：`GET /api/work/page/{current}/{size}`

| 参数 | 类型 | 必填 | 说明 |
| :--- | :--- | :--- | :--- |
| current | Long | 是 | 当前页码，从 1 开始 |
| size | Long | 是 | 每页数量，范围 1-500 |
| workTitle | String | 否 | 作品标题，支持模糊查询 |
| userId | Integer | 否 | 用户 ID，精确查询 |
| seriesId | Integer | 否 | 系列 ID，精确查询 |
| isOriginal | Boolean | 否 | 是否原创 (true/false) |

### 3.2 图像文件获取
**接口地址**：`GET /api/image/work/get?filePath={path}`

- **filePath**: 对应作品记录中的 `img_url` 字段。
- **配置位置**: `src/config/api.js` 中的 `WORK_IMAGE_API`。

## 4. 核心逻辑详解

### 4.1 数据转换流程
1. **API 层** (`workApi.js`): 获取原始 JSON，提取 `records`。
2. **转换层**: 遍历 records，利用 `getWorkImageUrl` 拼接完整图片路径。
3. **算法层**: 
   - 随机分配基础高度（260-460px）。
   - 执行列尾自适应（剩余空间不足 350px 时自动填满）。
   - 执行周期性高度交换（打破规律感）。

### 4.2 Composable 状态管理
`useWorkWaterfall` 返回以下响应式对象：

| 属性/方法 | 类型 | 说明 |
| :--- | :--- | :--- |
| `waterfallImages` | Ref\<Array\> | 转换后的图片数据数组 |
| `isLoading` | Ref\<Boolean\> | 加载状态 |
| `error` | Ref\<String\> | 错误信息 |
| `hasMore` | Ref\<Boolean\> | 是否还有更多数据 |
| `loadWorks(options)` | Function | 核心加载方法，支持 `reset` 和 `filters` |
| `loadMore()` | Function | 加载下一页（追加模式） |
| `refresh(filters)` | Function | 刷新数据（重置到第一页） |

## 5. 常见问题 (FAQ)

**Q: 为什么图片不显示？**
A: 请检查浏览器 Network 面板，确认 `/api/image/work/get` 请求是否返回了正确的图片二进制流。同时确认后端已开启跨域（CORS）。

**Q: 如何实现无限滚动？**
A: 监听容器的滚动事件，当触底时调用 `loadMore()` 方法即可。

**Q: 如何修改每页加载数量？**
A: 在 `useWorkWaterfall.js` 中修改 `pageSize.value`，或在调用 `loadWorks` 时传入自定义逻辑（目前默认 50 条）。
