# 管理员端前端技术总结

## 1. 核心框架

| 技术 | 版本 | 用途 |
|------|------|------|
| Vue 3 | 3.5.30 | 前端核心框架，Composition API + `<script setup>` |
| Vue Router | 5.0.3 | 客户端路由，含路由懒加载与导航守卫 |
| Vite | 7.3.1 | 构建工具与开发服务器 |
| pnpm | — | 包管理器 |

---

## 2. 架构设计：三层分离

```
src/config/api.js        →  API 地址配置中心（唯一来源）
src/api/adminApi.js      →  纯 HTTP 请求层，无 Vue 依赖
src/composables/useAdmin.js →  业务逻辑、状态管理、表单验证
src/views/Admin.vue      →  仅 UI 渲染与事件绑定
```

### 各层职责

| 层 | 文件 | 职责 |
|----|------|------|
| 配置层 | `src/config/api.js` | 定义 `ADMIN_API` 对象，集中管理所有管理员接口地址 |
| API 层 | `src/api/adminApi.js` | 封装 `fetch` 调用，Token 注入，返回 `{ success, data, message }` 统一格式 |
| Composable 层 | `src/composables/useAdmin.js` | 管理所有模块的响应式状态、分页、CRUD 业务逻辑 |
| 视图层 | `src/views/Admin.vue` | 组合子组件，解构 composable，绑定事件 |

## 3. 组件化拆分

### 目录结构

```
src/components/componentsAdmin/
├── AdminTopbar.vue      # 顶部导航栏（Logo、返回首页、退出登录）
├── AdminSidebar.vue     # 侧边导航栏（Tab 切换）

├── AdminAuditTab.vue    # AI 审核记录 Tab
├── AdminChangeTab.vue   # 用户数据变更审核 Tab
└── AdminLogTab.vue      # 操作日志 Tab
```

### 通信模式

- **单向数据流**：父组件 `Admin.vue` 持有所有状态，子组件通过 `props` 接收数据 + `emits` 回传事件
- 所有业务逻辑集中在 `useAdmin.js` composable 中，子组件不包含业务逻辑

## 4. 状态管理

### 不使用 Pinia Store

管理员端所有状态通过 composable (`useAdmin.js`) 内部的 `ref()` / `reactive()` 管理，无需全局 Store。

### 状态分类

| 模块 | ref 状态 | 数量 |
|------|----------|------|
| 用户管理 | `userList`, `userLoading`, `userKeyword`, `userTotal`, `userRoleFilter`, `userStatusFilter`, `userOrderBy` | 7 |
| 作品管理 | `workList`, `workLoading`, `workKeyword`, `workTotal`, `workOrderBy`, `workApprovalFilter` | 6 |
| 评论管理 | `commentList`, `commentLoading`, `commentKeyword`, `commentTotal`, `commentOrderBy`, `commentApprovalFilter` | 6 |
| 审核记录 | `auditList`, `auditLoading`, `auditTotal`, `auditKeyword`, `auditContentTypeFilter`, `auditApprovalFilter`, `auditOrderBy` | 7 |
| 变更审核 | `changeList`, `changeLoading`, `changeTotal`, `changeTypeFilter` | 4 |
| 操作日志 | `logList`, `logLoading`, `logTotal`, `logKeyword`, `logOrderBy` | 5 |

### 弹窗状态

## 5. HTTP 通信

### 技术选型

- **仅使用浏览器原生 `fetch` API**，**禁止 axios**
- 请求格式根据接口类型选择：
  - `POST` + `application/x-www-form-urlencoded`（列表查询、增删改操作）
  - `GET` + URL 查询参数（分页列表查询）

### 鉴权方式

```javascript
// Bearer Token 模式
headers: { 'Authorization': `Bearer ${token}` }
// Token 存储在 localStorage
localStorage.getItem('token')
```

### 响应处理

- 兼容 `result.code` 和 `result.recode` 两种字段名
- API 层统一返回 `{ success: boolean, data?: any, message?: string }`

### 参数编码

- POST 请求使用 `URLSearchParams` 构造表单数据（自动编码）
- GET 请求使用 `encodeURIComponent()` 手动编码查询参数

## 6. UI 交互组件

### 自定义工具弹窗

| 工具 | 文件 | 技术实现 |
|------|------|----------|
| 通知 Toast | `src/utils/notification.js` | `createApp` + `h()` 动态挂载 `NotificationToast.vue` |
| 确认对话框 | `src/utils/confirmDialog.js` | `createApp` + `h()` 动态挂载 `ConfirmDialog.vue`，返回 `Promise<boolean>` |

### 内联弹窗

创建用户和修改标题的弹窗使用了条件渲染（`v-if`）+ `.ad-overlay` 遮罩层方式，样式定义在 `admin.css` 中。

### 内联 SVG 图标

所有图标均使用内联 `<svg>` 元素（Heroicons 风格），无外部图标库依赖，图标路径通过 `navItems` 数组传入侧边栏。

| 动画 | 实现 |
|------|------|
| 骨架屏闪烁 | `@keyframes ad-shimmer` + `linear-gradient` 移动背景位置 |

## 8. 布局方案

```
┌──────────────────────────────────────────┐
│  AdminTopbar (sticky, h=56px)            │
├────────┬─────────────────────────────────┤
│ Sidebar│  AdminUserTab                   │
│ 220px  │  AdminWorkTab                   │
│        │  AdminCommentTab      (动态渲染) │
│        │  AdminAuditTab                  │
│        │  AdminChangeTab                 │
│        │  AdminLogTab                    │
└────────┴─────────────────────────────────┘
```

- **顶部栏**：`position: sticky; top: 0; z-index: 100`
- **主体**：`display: flex`，侧边栏固定宽度，内容区 `flex: 1; min-width: 0`
- **Tab 切换**：使用 `v-if` 按需渲染（非 `v-show`），首次切换时自动加载数据

## 9. 分页策略

采用 **"加载更多"** 模式（非传统分页器）：

```javascript
// 首次加载：{ reset: true } → 清空列表，page=1
// 加载更多：{ reset: false } → 追加数据，page++
// 判断是否还有更多：list.length < total
```

- 每页 20 条
- 加载中锁定（`loading` 为 true 时阻止重复请求）
- 底部显示"加载中" spinner 或"加载更多"按钮或"— 已加载全部 —"

## 10. 路由配置

```javascript
{
  path: '/admin',
  name: 'admin',
  component: () => import('../views/Admin.vue'),  // 懒加载
  meta: { requiresAuth: true },                    // 需要登录
}
```

- 使用路由懒加载减少首屏体积
- 全局前置守卫检查 `meta.requiresAuth`，无 Token 重定向到 `/login`

## 12. 调试日志规范

在 `adminApi.js` 中每个接口调用前后均有详细日志：

```javascript
console.log('📋 获取用户列表:', url, '参数:', { page, size, ... })
console.log('用户列表响应:', JSON.stringify(result, null, 2))
console.log('✅ 用户列表获取成功')
console.error('❌ 用户列表获取失败:', result.message)
```

使用 emoji 前缀区分日志类型：`📋` 请求、`✅` 成功、`❌` 失败、`🔒` 封禁、`🗑️` 删除、`👤` 创建、`🔑` 密码、`📝` 审核、`✏️` 编辑、`💬` 评论。

## 13. 技术亮点总结

1. **纯 Vue 3 Composition API**：无 Class 组件、无 Options API 混用
2. **零外部 HTTP 库**：全程使用 `fetch`，减少依赖体积
3. **零图标库依赖**：全部使用内联 SVG，按需定制
4. **轻度状态管理**：无需 Pinia Store，composable 内聚性高
5. **命令式弹窗**：`createApp` + `h()` 动态挂载，组件树解耦
6. **骨架屏加载**：纯 CSS 动画实现，无额外依赖
7. **完整的错误处理**：API 层 try-catch → Composable 层业务判断 → 视图层 Toast 提示
